## Global Constraints

These bind **every** task. Copied verbatim from the spec/backlog and session standing rules.

**Both repos:**
- **Never `git add -A`.** Stage only the files a task creates/modifies. The repo root has pre-existing untracked files (`COurses-handoff/`, `content/`, `docs/COURSE_PLATFORM_BLUEPRINT.md`, `scripts/*.sql`, `scripts/push-prod-schema.ts`, `*.md` legal files) that MUST NOT be committed.
- **Do not deploy, push to `main`, or touch prod** during task execution. Deployment is a separate owner-gated step (see "Deployment" at the end).
- **Bash:** never end a command with `pkill`/`kill` (exit 144 kills the shell). Leave dev servers running or kill them in a separate command.

**COURSES (devince):**
- Schema changes go through migrations only. `push:false` is set and must stay false. Each model change = `DATABASE_URI=<dev-uri> pnpm payload migrate:create <name>` + commit the generated migration. Never restore `push:true`.
- After any collection/block config change run `pnpm generate:types`.
- Tests: `pnpm test:int` (Vitest, `src/**/*.test.ts`, node env, mock `fetch` via `vi.stubGlobal`). Build: `pnpm build`.
- **Consent gate is server-side and authoritative:** the checkout route MUST reject with HTTP 400 unless `consent === true`. `withdrawalConsentAt` is ALWAYS server-stamped (`new Date().toISOString()`), never read from the client.
- **safeNext rule:** a redirect `next` value is honored only if it is a same-origin RELATIVE path — starts with a single `/`, not `//`, no backslash, no whitespace/control chars, no `scheme:` prefix. Reuse the existing `safeNext` from `src/app/courses-app/login/LoginForm.tsx`.
- Brevo email is best-effort: an email failure must NOT fail the Stripe webhook (the access grant already succeeded).

**NDQS (courses-platform):**
- Alembic migration chain head is `006` (`backend/alembic/versions/006_enrollment_revoked.py`, `down_revision="005"`). Any new migration chains after `006` as `007`.
- Tests: pytest with `asyncio_mode=auto` (`backend/pyproject.toml`). Run with `backend/.venv/bin/python -m pytest tests/<file> -v` (run from the `backend/` dir). DB mocked via `AsyncMock()`; endpoints driven via `httpx.AsyncClient(transport=ASGITransport(app=app))`; deps overridden via `app.dependency_overrides[get_db]`.

- Redis is reached via `from app.redis import get_redis` (`await get_redis()` → `redis.Redis`, `decode_responses=True`). In prod it is real Redis; in dev `memory://` for the limiter.
- Frontend tests: Vitest + jsdom (`frontend/vitest.config.ts`, `npm run test` = `vitest run`). The frontend API base is `process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000"`.
- Email failures must stay best-effort (already are): they are logged/recorded, never raised to the caller.

```ts
export async function DELETE(req: NextRequest, { params }: { params: Promise<{ idOrSlug: string }> }) {
  const auth = requireBearer(req) // reuse the same guard PATCH uses
  if (auth) return auth
  const { idOrSlug } = await params
  const payload = await getPayload({ config: configPromise })
  const resolved = await resolveDocId(payload, 'program', idOrSlug, 'pl')
  if (typeof resolved !== 'number') return NextResponse.json(resolved, { status: 404 })
  await payload.delete({ collection: 'program', id: resolved, overrideAccess: true })
  return NextResponse.json({ deleted: true, id: resolved })
}
```
