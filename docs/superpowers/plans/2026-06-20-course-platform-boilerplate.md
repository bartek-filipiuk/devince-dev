# Course-Platform Boilerplate Product Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a clean, genericized, sellable boilerplate (the full devince course + digital-products platform minus NDQS) in a separate repo, with installer, docs, feature list, license, and landing copy, packaged as a zip ready to list on apps.devince.dev.

**Architecture:** Snapshot devince's tracked tree into a fresh repo via `git archive` (auto-excludes untracked junk + `.git` + `.env`), then strip NDQS, de-brand to env-driven `NEXT_PUBLIC_SITE_NAME`/URLs, replace seed with a neutral demo, delete internal docs, write buyer-facing docs, verify (build + tests + fresh-seed + a hard leak-scrub grep), and zip. The live devince repo is never modified.

**Tech Stack:** Next.js 15, Payload CMS 3.67, PostgreSQL, pnpm, vitest, Docker. (No new dependencies.)

## Global Constraints

- **The live `devince.dev` repo is NEVER modified.** ALL work happens in the copy at `STARTER=/home/bartek/main-projects/course-platform-starter`. Every task `cd "$STARTER"` and commits there. After Task 1, confirm `git -C /home/bartek/main-projects/devince.dev status` shows no new tracked changes from this work.

No `.env`, no `node_modules`, no devince `.git` history in the zip.
- **License (verbatim, used in Task 6 LICENSE + Task 8 LANDING):** commercial, **unlimited own projects** (personal + commercial), **no resale/redistribution of the boilerplate source itself** (original or modified) as a competing product/template.

Hardcoded prod URLs → `NEXT_PUBLIC_COURSES_URL` / `NEXT_PUBLIC_APPS_URL` (fallback `https://courses.example.com` / `https://apps.example.com`). Email fallbacks → `support@example.com`.
